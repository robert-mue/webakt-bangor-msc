                  File

                  As in AKT5, this menu does very little, and does not have the file commands one is used to in most applications - e.g. for opening and closing files, printing, etc. I have removed two redundant commands (), and added in a command for specifying your user name.

                  User Name

                  This command used for webAKT's auto-saving mechanism, which saves the current versio of the knowledge based each time you make a change to it. Simply enter your name in the dialog box, and it will be added as an extra identifier for the knowledge base.

                  Exit from AKT

                  This has been retained purely because it was in AKT5. However, web applications do not have the concept of exiting - yousimpy close the browser window or navigate to another web page - so all it does is to generate a message to that effect.

                  KB

                  This menu is a key one in AKT5 for working with a knowledge base, and includes operations (e.g. for opening and closing knowldege bases) which one would normally expect to fnd under the File menu. For consistecy with AKT5, I have retained AKT5's commands for this menu.

                  The main differences come to lightwhen you actually use the commands, but these are generally pretty minor. For example, there is no distinction between Save and Save As... in a web application, when saving to your local file system, since security considerations prevent JavaScript writing diretly to file. Instead, both go through a file-save dialog, like in a normal Save As... command.

                  New Kb...

                  Use this to create a new, empty knowledge base. (Discussions are currently (July 2021) in progress as to whether one needs to give a knwledge base a name, or whether it simply takes its name from the file it is saved in (as with most other applications in which yu can create and save a document).

                  Open Kb...

                  This command enables you to open an already-created knowledge base which exists in your local file system (i.e. on the computer you are using). This file could be one that you have created and saved yourself, or than has been gven to you by someone else.

                  Like AKT5, webAKT knowledge bases are saved as text files, so can be read and edited in a normal text editor. There is a very close correspndence between the two, though AKT5's format is slightly more human-readable (not that you should really need to do this.

                  Save Kb...

                  This saes a knowledge base in a file on your computer. Unlike a normal Save cmmand, you are presented with a file-save dialog, and neeed to browse to the appropriate directory and give the file a name. As webAKT uses JSON format, the file should hve the extension .json (in contrast with AKT5, whch save knowlefge bases in Prolog syntax, but used the extension .kb).

                  Save Kb As...

                  This behaves exactly the same as the Save KB command described above.

                  Close Kb >

                  This command allows you to close (remove) a knowledge base from the current use of webAKT. If only one knowledge base is currentky open, then the comamnd is acted upon omediately, and it s renoved (leaving webAKT with no knowledge abses). If more than one knowedge base s open, then you get a submenu allowing you to chose thene you want to close.

                  All refeneces to the close d knpwpedge base are removed.

                  Select Kb >

                  Like aKT5, webAKT has the concept of a "current knowledge base". This saes you from having to repatedly choose a knowledge base when undertaking some operation or analysis. This command allows you to select another knowledge abse (if more than one are currently open),which then becomes the "current knowedge base".

                  Freeze Open Kb >

                  This command prevents you from making any changes to a knowlege base (e.g. to stop you accidentally making a change to a standard, curated knowledge base). If there is more tnan one currently open, you are presented with a submenu which allows you to chose the one you want to freeze.

                  Memos >

                  In AKT5, a "memo" is a small amount of plain text giving extra information about one of the named components. For example, a statement memo provides extra information about a particular statement (not, to avoid confusion, about the set of all statements).

                  The one exception to this is the so-called "Welcome" memo, which actually provides several, pre-defined, types of information (metadata) about a knowledge base. Unlike the other memos, this information is organised into separate fields - e.g. Title, Author, Study Area, etc. The information is presented automatically when you open a saved knowledge base, but otherwise has to be found by going to Memos > Welcome, and selecting the desored knowledge base.

                  It seems to me that this approach goes beyond a simple memo, and does not give the prominence to a knowledge bases's metadata which it deserves. I have therefore added an extra item to the KB menu (KB Metadata), and disabled the KB > Memo > Welcome command.

                  As a technical aside, I have also slightly modified the format for a knowlefge base file. The knowledge base metadata is now a top-level object within the file, with separate properties for each of the metadata fields, ratherthan the plain text entry which is difficut to parse used in the AKT5 file format.

                  Welcome

                  See note for Memos, above. This Memo option has been replaced by the command "KB Metadata" in the KB menu.
